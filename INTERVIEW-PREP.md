# BA Interview Preparation Plan

## Phase 1 — Template Mastering (3-5 days)

Build real artifacts based on Embafinans Credit Scoring project.

### BRD (Business Requirements Document)
- [ ] Write BRD using Babok v3 template structure
- [ ] Project: Credit Scoring & Pre-Screen Risk Assessment
- [ ] Sections: Business goals, stakeholders, scope, high-level requirements, assumptions, constraints

### FRD (Functional Requirements Document)
- [ ] Extract functional requirements from BRD
- [ ] Use REQ-101 numbered format
- [ ] Cover: Credit scoring workflow, risk assessment rules, integration points

### User Stories with Gherkin Acceptance Criteria
- [ ] Write 10 real user stories from Embafinans projects
- [ ] Each with Given/When/Then format
- [ ] Jira-compatible format

### BPMN As-Is / To-Be
- [ ] As-Is: Manual credit scoring process before automation
- [ ] To-Be: Automated multi-factor assessment workflow
- [ ] Tool: draw.io — export as PNG
- [ ] Include swimlanes (risk, sales, operations, IT)

### Sequence Diagrams
- [ ] Credit application → scoring engine → decision flow
- [ ] Payment gateway integration flow
- [ ] Partner API onboarding flow
- [ ] Tool: draw.io — export as PNG

### Gap Analysis
- [ ] Current system vs new scoring system
- [ ] Identify gaps in: data, process, technology, people

### Swagger / OpenAPI 3.0
- [ ] REST API spec for credit scoring endpoints
- [ ] Include request/response schemas

### RICE Prioritization Example
- [ ] Score 5 real requirements using RICE (Reach, Impact, Confidence, Effort)

## Common Interview Questions to Prepare

### Behavioral
- "Tell me about a time you had conflicting requirements from stakeholders"
- "How do you handle changing requirements mid-sprint?"
- "Describe a project that failed — what did you learn?"

### Technical
- "Walk me through your BRD structure"
- "How do you write acceptance criteria?"
- "Explain the difference between As-Is and To-Be analysis"
- "What is RICE and when would you use it vs MoSCoW?"
- "How do you validate requirements with stakeholders?"
- "What is your approach to UAT?"

### Scenario-Based
- "A stakeholder wants a feature that adds no business value — what do you do?"
- "Development team says a requirement is unclear — how do you clarify?"
- "Two departments disagree on process flow — how do you resolve?"
